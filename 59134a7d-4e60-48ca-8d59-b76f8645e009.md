# exocmd__Grounding_labelTemplate

`exo__StringProperty` на `exocmd__Grounding` (cardinality 0..1, range `xsd:string`).

Substitution-token template для производства `exo__Asset_label` нового instance в one-click flow (когда `inputSchema` отсутствует → нет модала → executor сам строит label из template).

## Pipeline

1. `CommandResolver.loadGroundingDefinition` парсит RDF triple → `grounding.labelTemplate: "<template-string>"`.
2. `executor.executeCreateInstance` доходит до `applyMissingScalarPrimitives`.
3. Если `userInput.label === undefined` (нет modal-input) AND `grounding.labelTemplate` set:
   - `substituteVariables(template, targetIRI, userInput, targetFm, targetFilePath)` → resolved string
   - Если resolved.trim().length > 0 → используется как `exo__Asset_label`
   - Иначе (empty/whitespace) → fallback на `"Untitled"`
4. Иначе → fallback на `userInput.label ?? "Untitled"` (pre-RFC behavior).

## Supported substitution tokens

| Token | Resolves to | Example |
|---|---|---|
| `$target` | targetIRI (full vault IRI) | `obsidian://vault/03 Knowledge/kitelev/deaa0051-…md` |
| `$target.<prop>` | target frontmatter property value | `$target.exo__Asset_label` → `Осознал, что делаю шелуху` |
| `$today` | `YYYY-MM-DD` | `2026-05-29` |
| `$todayStart` | `YYYY-MM-DDT00:00:00` (local midnight) | `2026-05-29T00:00:00` |
| `$nowLocal` | `YYYY-MM-DDTHH:mm:ss` (no ms, no tz) | `2026-05-29T10:04:02` |
| `$nowCompact` | `YYYY-MM-DD-HH-mm` (filename-safe, RFC ce27e55d) | `2026-05-29-10-04` |
| `$targetFolder` | parent folder of `$target` | `03 Knowledge/kitelev` |

## Background

Добавлено в RFC `ce27e55d-e5e6-4b52-86ba-3ac9b30de38a` — одноразовое расширение TBox interpretation infrastructure для one-click create_instance flow.

Plugin PR: [kitelev/exocortex#3287](https://github.com/kitelev/exocortex/pull/3287) (merged 2026-05-29 как `de9ae418`).

## Sibling / related properties

- [[f1fa0280-bd3b-4e3d-ace3-7e36c80127c6|exocmd__Grounding_prefillLabelWithDate]] — RELATED но **disjoint** концепт: prefillLabelWithDate работает с **inputSchema modal** (pre-fills default value of label field, user может override). labelTemplate работает **без modal** (executor сам строит label без user input).
- `exocmd__Command_openInSameTab` (UID a7262f21-5fa0-4bae-96c2-7e9a16c0c054) — sibling property на Command для navigate-in-active-leaf semantics.
- [[025fafe0-a42b-4b39-b917-ae8c3b2b1f59|exocmd__Command_confirmMessage]] — required null/empty для true one-click flow (иначе confirm dialog тормозит interaction).

## Usage pattern (canonical)

Для one-click logging кнопки на prototype-instance (например supervision/observation/check-in patterns):

```yaml
# Command:
exocmd__Command_openInSameTab: true
# Confirm message: НЕ устанавливать (omit field → skip dialog)
# Success message: можно установить — отображается как notification

# Grounding (type=create_instance):
exocmd__Grounding_labelTemplate: "$target.exo__Asset_label $nowCompact"
# inputSchema: НЕ устанавливать (omit field → skip modal)
# prefillLabelWithDate: НЕ устанавливать
```

Результат: click → create instance с `exo__Asset_label: <prototype label> YYYY-MM-DD-HH-mm` → open в той же вкладке. Zero friction.
