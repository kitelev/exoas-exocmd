# exocmd__TokenInvocation

RFC 727572d2. Parameterised SubstitutionToken invocation. Bridges `PropertyDefault_value` (which accepts a single asset ref) and parameterised resolvers that need a runtime argument.

## Properties (Phase 1)

- `exocmd__TokenInvocation_token` — ref к `exocmd__SubstitutionToken` instance. Range: SubstitutionToken IRI. Cardinality 1.
- `exocmd__TokenInvocation_parameter` — string argument passed to the resolver. Range: xsd:string. Cardinality 0..1 (parameterless tokens omit).

## Usage shape

A TokenInvocation instance has the following frontmatter shape:

- `exo__Instance_class` includes the TokenInvocation class UID
- `exocmd__TokenInvocation_token` references a SubstitutionToken instance
- `exocmd__TokenInvocation_parameter` literal string parameter (optional)

A PropertyDefault references this TokenInvocation via its `exocmd__PropertyDefault_value` wikilink. Parameterless tokens (e.g. `$today`) continue to be referenced directly as bare SubstitutionToken instances — backward-compatible.

## Engine semantics

Resolver detects: when PropertyDefault_value asset's `exo__Instance_class` includes the TokenInvocation class → load `_token` ref + `_parameter` literal, dispatch via SubstitutionResolverRegistry passing parameter. Else (asset is bare SubstitutionToken) → dispatch by resolver-id without parameter.
