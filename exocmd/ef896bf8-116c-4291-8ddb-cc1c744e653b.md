# Create Instance grounding

`exocmd__Grounding` (type `create_instance`) for the homoiconic «Create Instance»
button (project bbe40f8c, T1).

The button is bound to the `exo__Class` metaclass, so the click target is always
a class-definition page. This grounding:

- `targetFolder = $isDefinedByFolder` — co-locates the new instance in the folder
  of its chosen `exo__Asset_isDefinedBy` ontology (co-location invariant),
  resolved at execution time by the injected `RefToFolderResolver`.
- `inputSchema` — a two-field form: `label` (text) + `exo__Asset_isDefinedBy`
  (fuzzy reference-picker over `exo__Ontology` instances,
  `targetClassUid = 829b9b3b-...`).
- `propertyDefault` → sets `exo__Instance_class` to the host's own UID via the
  `$targetClassSelf` token (the host IS the class). No `targetClass` is declared.

uid / createdAt / label / aliases come from the Universal Default Template
(`label = $userInputLabel`). T3 will add class-specific required properties to
the form.
