# Create Next Iteration grounding

The `create_instance` grounding behind the re-instated `ems__Task` «Create Next
Iteration» command (`267ceae0`, req `2d1ffced`). Clones the clicked **Done** task
into a fresh **Backlog** next iteration — the honest-success counterpart to
«Partially Done» (which is the failure path and additionally closes the source).
Unlike «Partially Done», this grounding is standalone (NOT a composite) and does
**not** mutate the source: it only creates the next iteration.

- **`targetClass`** → `ems__Task` (`1b20a8f0`); **`targetFolder`** →
  `$targetFolder` — co-locates the iteration with the original (by the source's
  `isDefinedBy`), replacing the decommissioned hardcoded `03 Knowledge/inbox` path
  (RFC 0b7a2fad co-location).
- **`labelTemplate`** → `$target.exo__Asset_label` — clones the title verbatim.
- **`cloneTargetBody: true`** — clones the original's markdown body forward
  (context notes carry across iterations; req 915b20b2 engine primitive).
- **InheritanceRules** — REUSES the «Partially Done» clone mechanism (req
  `87361a62`, by reference — the same shared leaf rules): copies
  `ems__Effort_area` (`9f9ede48`), `ems__Effort_parent` (`e9761329`),
  `ems__Task_zone` (`0f8c1d14`), `exo__Asset_relates` (`2b9004cf`),
  `ems__Effort_deadlineTimestamp` (`563f5c90`), `ems__Effort_timeEstimateMinutes`
  (`b0817fa9`), and sets `ems__Effort_prevIteration` → the source task
  (`6ebb59d0`, uid→prevIteration back-link — replaces the legacy `linkBackProperty`
  mechanism; UWI child→prev, reverse derived via SPARQL).
- **PropertyDefault** — REUSES `ce287f1a`: sets `ems__Effort_status` = Backlog
  (`753a44d5`). Fresh uid / createdAt / instance_class / isDefinedBy come from the
  Universal Default Template.

The next iteration deliberately carries **no** execution timestamps (start / end /
resolution) and **no** `ems__Effort_partiallyDone` flag — the IRs above only copy
substantive domain properties, and status is overridden to Backlog.
