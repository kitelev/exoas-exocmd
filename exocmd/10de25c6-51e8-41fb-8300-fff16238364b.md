# Is Done, not a prototype, and not archived (Create Related Task)

**AllPrecondition** (AND) — the **inverse** (Done-only) gate for «Create Related
Task» (`62b8c49c`), for req(ems) `f6c2b98c-4882-47ed-a79d-2e56266e39d1`:

1. [[2bee97b7-da01-4309-925b-f605e7816ba1|Allow re-open (when status = Done)]] — status = Done AND not a prototype; AND
2. [[28f722ba-cd01-44c6-b69d-b1c59b0e14fd|Not archived]] — hides «Create Related Task» on an archived (Done + archived) task.

«Create Related Task» previously shared the not-a-prototype+not-archived
precondition (`6dd007b8`) with «Create Action». The target matrix puts the two on
**opposite** sides of Done — Create Action is Backlog+Doing only, while spawning a
follow-up from a **completed** task (Create Related Task) makes sense only once the
source is Done. So the two are split: Create Action → not-Done wrapper (`99da8e4b`),
Create Related Task → this Done-only wrapper. Reuses the same atomics that gate
«Re-open» / «Create Next Iteration» (2bee97b7 + 28f722ba), each now a leaf in ≥2
composites (non-vacuous). Matrix §1: Create Related Task = Backlog ⛔ / Doing ⛔ /
Done ✅ / Archived ⛔.
