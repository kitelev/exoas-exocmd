# exocmd__InheritanceRule

Declarative target-property mapping rule attached to `exocmd__Grounding` via `Grounding_inheritanceRule`.

## Properties (Phase 1)

- `exocmd__InheritanceRule_sourceProperty` — какое property взять у `targetIRI` (range: `exo__Property`, card 1)
- `exocmd__InheritanceRule_targetProperty` — под каким именем записать у нового instance (range: `exo__Property`, card 1)
- `exocmd__InheritanceRule_targetClassCondition` — apply только если target instance of (range: `exo__Class`, card 0..1)
- `exocmd__InheritanceRule_targetClassExclusion` — apply только если target НЕ instance of (range: `exo__Class`, card 0..N)
- `exocmd__InheritanceRule_priority` — numeric tie-break (range: integer, default 50)

## Engine semantics (resolution order)

1. Filter rules: keep where `_targetClassCondition` absent OR target instance of `_targetClassCondition`.
2. Filter out rules where target instance of any `_targetClassExclusion`.
3. Sort by `_priority` descending.
4. Apply: later writes override earlier for same `_targetProperty`.

## Idempotency

Idempotency key (Phase 2 migration): unique tuple `(Grounding-uid, sourceProperty-uid, targetProperty-uid, targetClassCondition-uid OR null)`.
