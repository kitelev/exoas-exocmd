# Start effort (composite)

Composite grounding mirroring v15.38.0 `start-effort` command behaviour:

1. Set `ems__Effort_status` → `[[ems__EffortStatusDoing]]`
2. Set `ems__Effort_startTimestamp` → current local timestamp

Atomic via `GroundingExecutor.executeComposite` rollback on step failure. Used by command `Set Status Doing` (`e941b3bb`).

Restores timestamp-write behaviour lost when UI commands migrated from hardcoded `StartEffortCommand` to vault-driven RDF assets.
