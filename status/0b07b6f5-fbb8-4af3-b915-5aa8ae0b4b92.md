# Mark Reviewed → Task binding

Binds "Mark Reviewed" command (`e42f2e7c`) to `ems__Task` assets. Phase 1.5 gap closure for RFC 31c1a0be — without this binding, the Mark Reviewed button would never appear on Task cards even after Phase 3/4 deletion of legacy `MarkReviewedCommand.ts`.

Order `140` places the button between Set Status Done (130) and Set Status Cancelled (150) — logical adjacency for a post-Done review action.

## Activation status

Same as the underlying Mark Reviewed Command (`e42f2e7c`): grounding uses wikilink-form `targetProperty`, which requires Phase 3 parser update. Until then, legacy `MarkReviewedCommand.ts` remains the active path (no functional regression).
