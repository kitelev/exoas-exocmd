# Is ontologically archived

SPARQL ASK precondition for the «Un-archive Ontologically» command — the inverse
of the forward has-archive-target gate
([[9b0fdc10-ccb9-46b0-80b8-d9a250a01c9b|Archived and not yet ontologically archived]]).
Returns `true` (command available) when the asset's current `exo__Asset_isDefinedBy`
is the **archive ontology** of some source ontology:

- `$target isDefinedBy ?arch . ?src Ontology_archiveOntology ?arch` — the asset
  currently lives in an ontology that is somebody's declared archive target, i.e.
  it was ontologically archived.

This is what makes un-archival **idempotent**: once restored, `isDefinedBy` points
at the source ontology (which is nobody's `exo__Ontology_archiveOntology` target
under the 1:1 pairing) → the ASK fails → the command is hidden / `apply` is a
no-op. No double relocation.
