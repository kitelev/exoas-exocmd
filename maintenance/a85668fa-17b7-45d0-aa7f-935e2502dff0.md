# Copy label to aliases via service

Grounding action that appends the value of `exo__Asset_label` (read via the `$target.<propertyName>` substitution introduced in Issue #3132) to the asset's `aliases` array property, with Set-based dedup.

Uses the declarative `property_append` grounding type — no TypeScript service code is involved. This is the Homoiconicity Invariant Q1 remediation for the formerly-`service_call`-based `copyLabelToAliases` pattern.

Behavior:
- Empty / missing `aliases` → `aliases: [<label>]`
- Existing `aliases` without the label → label appended
- Existing `aliases` already containing the label → no-op (idempotent)
- Missing `exo__Asset_label` on the target → grounding fails loudly with informative error
