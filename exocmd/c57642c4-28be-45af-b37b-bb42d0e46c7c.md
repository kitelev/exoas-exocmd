# Visible until terminal, not a prototype, not archived

**AllPrecondition** (AND composite) — available iff **all** children are true:
1. [[067e481d-ba2e-4022-a4ec-9e7741a0b7af|Visible until terminal (not Done, not Trashed)]] — the target effort is NOT in a terminal status (Done **or** Trashed); AND
2. [[5b49ef19-7eb9-4b67-bb4e-c9002537000b|Not a prototype]] — shared not-prototype guard (`NOT STRENDS(class,"Prototype")`), reused across the criticality + planning composites; AND
3. [[28f722ba-cd01-44c6-b69d-b1c59b0e14fd|Not archived]] — reusable archived-guard (`NOT EXISTS { $target exo:Asset_archived "true" }`).

**Purpose (D3, 2026-07-05).** The "Execute" command (`f86925c4`) originally used the bare `37517554` precondition, so its button **leaked onto `ems__ActionPrototype`** (a prototype has no Done status → visible-until-Done passed). Executing a *template* is meaningless — you create an instance and execute that. This composite adds the not-a-prototype guard so Execute renders only on real (non-prototype) actions, mirroring the task-prototype curation (planning buttons hidden). Behavior-preserving on regular actions (identical to `37517554` alone there).

⚠ Do **not** refactor `37517554` in place — it is shared with "Mark Done Post Factum". This composite isolates the guard so it can be applied selectively.

**Reuse (D4, 2026-07-06).** This composite is **semantically generic** (`AllPrecondition[Visible-until-Done, Not-a-prototype]` — nothing Execute-specific) and is now the shared guard for the **planning-command family** too: `plan-on-today`, `plan-for-evening`, `shift-day-backward`, `shift-day-forward`, `vote-on-effort`. They previously used the bare not-a-prototype guard (`5b49ef19`, or the equivalent atomic `847c37e0` for vote) with **no Done guard**, so they leaked onto completed (Done/terminal) Task/Meeting/Action efforts — planning/re-scheduling/voting on a finished effort is meaningless. Repointing them here adds the `Visible-until-Done` guard while preserving the not-a-prototype hiding (D1-D2). Behavior-preserving on active non-prototype efforts.

**Update (req `127763f8`, 2026-07-07).** Trashed is a **second terminal status equal to Done**. The `Visible-until-Done` conjunct (`37517554`) was replaced in place by the `Visible-until-terminal` NotPrecondition (`067e481d = Not[«Is terminal (Done or Trashed)» 9b85e89a]`) and the reusable `Not-archived` atomic (`28f722ba`) was AND-ed in — so all six consumers (Execute + the five planning/voting commands) now hide on Trashed and on archived, exactly like on Done. `37517554` itself is left untouched (it stays with «Mark Done Post Factum», which keeps its not-Done gate). Behavior-preserving on active non-prototype non-archived efforts.
