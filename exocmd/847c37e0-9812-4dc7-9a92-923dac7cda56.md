# Target is not a prototype

SPARQL ASK precondition: returns `true` when the target asset is **not** a
prototype-template — i.e. none of its `exo__Instance_class` values is a
prototype class (a class whose IRI local-name ends with `Prototype`:
`ems__TaskPrototype`, `ems__ProjectPrototype`, `ems__EffortPrototype`,
`exo__Prototype` itself, etc.).

**Why a name-suffix check, not the semantic `exo:Class_superClass*` walk**
(verify-before-assert, req 5579ffa1): in the real triple store a class's
`exo__Instance_class` resolves to the *symbolic* class IRI
(`https://exocortex.my/ontology/ems#TaskPrototype`), but `exo:Class_superClass`
triples are stored under the class *file* IRI (`obsidian://vault/<uid>.md`).
Those two representations are NOT connected by any triple (the command resolver
bridges them in code via label→uid→file lookup, which SPARQL cannot do), so a
transitive `$target exo:Instance_class/exo:Class_superClass* exo:Prototype`
path reaches only the symbolic class IRI and NEVER `exo:Prototype` → the gate
would never fire. The `STRENDS(…, "Prototype")` suffix check is queryable on the
symbolic IRI and catches every conventionally-named prototype class, including
future ones. A forget-guard test asserts every class that IS-A `exo__Prototype`
is named `…Prototype`, binding this heuristic back to the semantic intent.

Used to hide instance-lifecycle commands (Start / Mark Done / Set Criticality /
status transitions / child-creation / convert / vote) from prototype-template
assets while leaving universal-maintenance (`targetClass exo__Asset`) and the
prototype's own create-instance command untouched.

Realises requirement `5579ffa1-a829-4fcf-b93d-4fb664b02d62` (design node
`f853eadd-6132-43f8-84ba-25a75641096a`). The same `FILTER NOT EXISTS { … }`
clause is conjoined into the 17 existing lifecycle status/criticality/convert
preconditions so the not-a-prototype gate ANDs with their status gates.
