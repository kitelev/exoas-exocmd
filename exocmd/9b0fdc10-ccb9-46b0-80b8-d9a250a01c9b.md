# Archived and not yet ontologically archived

SPARQL ASK precondition for the «Archive Ontologically» command (RFC 78c2b7d0 C5).
Returns `true` (command available) when **both** gates hold:

- **Archived gate**: `$target exo:Asset_archived "true"` — the asset was flagged
  by stage-1 `toggle-archived` (the bare `archived: true` frontmatter flag is
  indexed as the unprefixed-whitelist triple `exo:Asset_archived "true"`,
  string literal — RFC 871c1e56 A-series).
- **Has-archive-target gate**: `$target isDefinedBy ?onto . ?onto Ontology_archiveOntology ?a`
  — the current ontology declares an archive ontology.

The has-archive-target gate is what makes ontological archival **idempotent**:
once relocated, `isDefinedBy` points at the archive ontology, which declares no
`exo__Ontology_archiveOntology` of its own → the ASK fails → the command is
hidden / `apply` is a no-op. No double relocation, no double postfix.
