# Create MeetingPrototype instance

Grounding для создания `ems__Meeting` instance из `ems__MeetingPrototype` (MeetingPrototype variant of universal Command `[[bb00efed-7b17-42f5-a2c4-7cadf3e0ab36]]`).

Picker (CommandResolver.loadLinkedGrounding) выбирает это grounding среди 5 у `bb00efed` потому что `Grounding_targetPrototype` matches binding `[[22093ca1-1b6f-477f-a138-d13043b04a97]]`'s targetClass = `ems__MeetingPrototype`. Требует exocortex plugin ≥ v16.27.3 (PR #3266 — multi-grounding picker by targetPrototype).

- `Grounding_type=create_instance` → `GroundingExecutor.executeCreateInstance`.
- `targetClass=ems__Meeting` → `exo__Instance_class: [[1b0a5e34-dd7f-4ead-b43a-6c7c5a5ecaca]]` в новом файле.
- `targetPrototype=7ab483c7-...|ems__MeetingPrototype` → используется picker'ом + executor пишет `exo__Asset_prototype: [[<triggering-asset-uid>]]` back-link.
- `targetFolder=$targetFolder` — резолвится в folder триггер-ассета (Issue #3136). Meeting создаётся рядом с prototype's siblings.
- `inputSchema` — modal с полем label (Meeting name, prefilled с датой через `prefillLabelWithDate: true`).

Inheritance: только `cbe000c4` (isDefinedBy unconditional). Принцип prototype = virtual aggregator — area / relates / прочие shared properties остаются на prototype; instance ссылается через `exo__Asset_prototype` back-link.
