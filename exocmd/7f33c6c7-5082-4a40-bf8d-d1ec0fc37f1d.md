# Clear aliases before re-mirroring label (#3798)

`property_delete` step for the "Set Label" composite. Removes the entire
`aliases` list so the following `property_append` step re-mirrors ONLY the new
canonical label — preventing stale-alias accumulation on repeated relabels
(Issue #3798). Composite rollback restores the original file if any later step
fails, so the clear is safe.
