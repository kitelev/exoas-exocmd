# Is terminal and not archived

**AllPrecondition** (AND composite) — available iff **both** children are true:
1. [[9b85e89a-8162-492f-b85f-2f8a1f21dded|Is terminal status (Done or Trashed)]] — the target effort is in a terminal status; AND
2. [[28f722ba-cd01-44c6-b69d-b1c59b0e14fd|Not archived]] — the shared archived-guard (`NOT EXISTS { $target exo:Asset_archived "true" }`).

The **«Archive Completed»** command (`52c88678`) precondition, replacing «Is Done
and not archived» (`8762ddc2`) — req(ems)
`127763f8-b97d-472f-872d-7c21ecdaacf6`. Any **terminal** (Done **or** Trashed)
effort that is not yet archived can be archived; previously only literal-Done was
archivable, so a Trashed task could never be archived through the UI. Reuses the
shared «Not archived» atomic (`28f722ba`, also used by the archived-gate family)
and the reusable «Is terminal» atomic (making «Is terminal» non-vacuous — a leaf
here and, negated, in «Visible until terminal»).
