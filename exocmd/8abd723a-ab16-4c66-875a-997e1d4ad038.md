# Mark Done Post Factum

Sets `ems__Effort_status` to Done ([[7b9b3116-7c3c-438c-9618-94fe301320a6]])
and **nothing else** — no `ems__Effort_endTimestamp`, no
`ems__Effort_resolutionTimestamp`. For logging work that was actually finished
earlier ("задним числом"): stamping the click moment would fabricate false
timestamps, so the command deliberately writes none.

- **Grounding**: `bdf3c81f` "Set status to Done" (reused single property_set
  step — the same step 1 of the Mark Done composite `77be16d6`, WITHOUT its
  timestamp steps 2-3).
- **Precondition**: `37517554` "Visible until Done" — one tap from ANY
  non-Done status (post-factum tasks often were never started in the
  tracker), unlike "Mark Done" (`923520d1`, Doing-only).
- **Icon**: history (Lucide) — visually distinct from Mark Done's
  check-circle.
- **Category**: status. **CLI**: `apply mark-done-post-factum <path>`.

Contrast with siblings: `923520d1` Mark Done = status + endTimestamp +
resolutionTimestamp (Doing→Done); `f86925c4` Execute (ems__Action) = status +
resolutionTimestamp (any→Done). This command = status only.
