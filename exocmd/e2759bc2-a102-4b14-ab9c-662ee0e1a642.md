# Has non-UID filename outside whitelist

SPARQL ASK precondition: returns `true` when the target asset's filename does not match its `exo__Asset_uid` AND the target is **not** an instance of a whitelisted label-naming class.

Whitelist (per BehavioralRule `152e39df-cc1b-4175-a4f2-2c2913018937` in vault-exodev aiKnow):

- [[2b754a16-cccf-49ee-af69-a18c6d1e3b63]] — `period__Week` (filename `YYYY-Www.md`, same plugin) — **symbolic IRI** (class has label)
- [[a298703f-3113-44fa-acf9-bc7d02d67040]] — `period__Day` (filename `YYYY-MM-DD.md`, 77 real daily-note assets) — **literal-string wikilink** `"[[a298703f-...]]"` (class has NO label → parser preserves raw wikilink instead of emitting IRI)

Used by the **Rename to UID** command ([[d0a0663b-407b-45bc-8c5c-e0ddfa883ba4]], RFC Phase 6 switch) to skip whitelist instances during bulk-rename.

VALUES IRI form — **mixed**: classes with `exo__Asset_label` (e.g. `pn__DailyNote`, `period__Week`) emit symbolic `<https://exocortex.my/ontology/<prefix>#<LocalName>>`; classes without label (e.g. `period__Day` UUID-named without label property) preserve raw wikilink as **string literal** `"[[<uid>]]"`. SPARQL VALUES accepts both forms — match by exact equality on object term. Verified 2026-05-23 follow-up #1 (RFC 1ce2a226 Phase 7 incident).

RFC 1ce2a226 §S1/M10 (URL-encoded UUID-canon) **superseded** — STEP H empirical verification 2026-05-23 returned 0 rows for UUID-canon form vs 290 rows for symbolic DailyNote.

Whitelist expansion: edit `VALUES ?wlCls { ... }` block in this asset's frontmatter — no code change required (homoiconicity Q1).
