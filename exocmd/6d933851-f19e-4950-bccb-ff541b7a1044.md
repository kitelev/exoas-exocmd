# exocmd__Grounding_targetValueQuery

`exo__ObjectProperty` on `exocmd__Grounding` (cardinality 0..1).

Read-side value-source for `property_set` groundings (RFC `78c2b7d0` C4). YAML serialization: `targetValueQuery: "[[<query__NamedQuery-uid>]]"`.

The referenced `query__NamedQuery` runs read-only (auto-injected `$currentAsset` = the target asset) and its scalar result becomes the `property_set` value — IRI scalar → `"[[<name>]]"` wikilink, literal → as-is. The read side computes a write-input: textbook CQRS.

## Disjointness axiom

Disjoint with `exocmd__Grounding_targetValueRef`, `exocmd__Grounding_targetValueLiteral`, and `exocmd__Grounding_targetValueSubstitution`. TBox cardinality: exactly one of the four value-sources set per `property_set` grounding (executor-enforced, `GroundingExecutor.executePropertySet`).

## Consumer

C5 «Archive Ontologically» — `property_set isDefinedBy` where the value is the archive ontology computed by a NamedQuery 1-hop (`currentOntology → exo__Ontology_archiveOntology`). Generic / reusable beyond archive.
