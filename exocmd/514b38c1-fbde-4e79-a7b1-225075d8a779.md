# Resolve source ontology

NamedQuery for the «Un-archive Ontologically» command — the symmetric inverse of
the forward «Resolve archive ontology» ([[5e2210f4-ab44-402c-b1b7-bd4dfb364301|Resolve archive ontology]]).
Computes the SOURCE ontology of `$currentAsset` by **reversing** the dedicated
`exo__Ontology_archiveOntology` relation — find the `?src` whose archive ontology
equals `$currentAsset`'s current `isDefinedBy` (the archive ontology it currently
lives in). Consumed read-side by the `property_set` grounding's `targetValueQuery`
(C4 CQRS bridge); `$currentAsset` is auto-injected with the click/apply target IRI.

Correct under the documented 1:1 archive-pairing (concept e15d22ae) — each archive
ontology is the `archiveOntology` of exactly one source, so the reverse yields
exactly one `?src`.

```sparql
SELECT ?src WHERE { $currentAsset <https://exocortex.my/ontology/exo#Asset_isDefinedBy> ?arch . ?src <https://exocortex.my/ontology/exo#Ontology_archiveOntology> ?arch }
```
