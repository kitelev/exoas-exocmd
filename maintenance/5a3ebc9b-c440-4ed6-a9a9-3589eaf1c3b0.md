# Duplicate current asset

Single-click clone of the currently open Exocortex asset. Creates a byte-verbatim copy in the same folder with two surgical frontmatter substitutions:

- `exo__Asset_uid` → freshly generated UUID v4
- `exo__Asset_createdAt` → current local ISO timestamp (Asia/Almaty)

Everything else (label, modifiedAt, instance class, all relations/prototype/parent, class-specific properties, the markdown body including filled checklist items and SPARQL blocks) is preserved unchanged. The new file is named `<new-uid>.md` per the UUID-canon rule and opened in a new tab with focus.

- **Precondition**: [[4cffa221-85e1-4e03-8f4e-af191aea3779|Has UID filename]] — palette entry is hidden when the active file isn't itself a UUID-canon asset (daily/weekly notes via `pn__DailyNote` / `period__Week` whitelist, ontology TBox label-named files, freeform `.md` without frontmatter)
- **Grounding**: [[1c49a105-ec80-4387-bac8-dc51e10d431c|Duplicate asset via service]] — invokes `duplicateAsset` registered service
- **Icon**: copy (Lucide)
- **Category**: maintenance
- **Palette**: registered in Obsidian Command Palette via `ExocmdCommandPaletteRegistrar` extended in Issue #3292. `paletteId: "duplicate-current-asset"` is the stable id for user hotkey bindings — must NOT change without a migration.

## Motivation

Recurring-work templates (weekly review checklists, supervision prototypes) and "fork this as starting point" workflows otherwise require 5-7 manual steps: open target, copy content, create new file, generate UUID, replace uid/createdAt, place near original. One Command Palette entry, one click.

Source: Issue [kitelev/exocortex#3292](https://github.com/kitelev/exocortex/issues/3292).
