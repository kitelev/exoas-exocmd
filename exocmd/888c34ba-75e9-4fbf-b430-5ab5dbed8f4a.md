# Label not yet in aliases

SPARQL ASK precondition that is **true** (button shown) when the target asset has an `exo__Asset_label` **and** that label is **not** already present among its `exo:Asset_aliases`. Returns **false** (button hidden) once the label has been copied into aliases — restoring, homoiconically, the visibility gate of the former TS `canCopyLabelToAliases`.

- Depends on `exo:Asset_aliases` emission (RFC 1b1c21b2 D1 — `aliases` whitelisted in `NoteToRDFConverter`).
- Used by the "Copy Label to Aliases" command (`a04a9129`) via `exocmd__Command_precondition`.
- **NO-LABEL note:** assets without a frontmatter `exo__Asset_label` get a synthesised basename label (converter fallback), so the ASK binds `?label` to the basename → button stays visible (basename ∉ aliases). Accepted behaviour, not strict `canCopyLabelToAliases` parity.
