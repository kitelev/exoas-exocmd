# Create action

Grounding bound to the "Create Action" command (`48a70029`). Creates a new
`ems__Action` via the declarative `create_instance` pipeline — full sibling of
the "Create task" grounding (`a222094b`), only the target class differs
(`ems__Action` instead of `ems__Task`).

- `targetClass` → `ems__Action` (`6a99d2ca`).
- PropertyDefault `d9aa9bb8`: new Action starts as `ems__EffortStatusDraft`
  (same default as Create Task).
- InheritanceRules `3f08f5a8` / `43731bae` / `cbe000c4` (reused, class-agnostic):
  target's `exo__Asset_uid` → `ems__Effort_area` (Area target) or
  `ems__Effort_parent` (Project/Effort target), plus `exo__Asset_isDefinedBy`
  pass-through. So "Create Action" invoked on a Project sets the new Action's
  `ems__Effort_parent` to that project (epic fixture `6528ecfa`).
