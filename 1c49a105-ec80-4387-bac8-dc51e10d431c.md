# Duplicate asset via service

Grounding action that invokes the `duplicateAsset` registered service.

Behaviour (`packages/services/src/grounding-service-factories.ts:createDuplicateAssetService`):

1. Resolves the active file from `targetIRI` (`obsidian://vault/<encoded-path>`)
2. Reads the file content via `vaultAdapter.read`
3. Generates a fresh UUID v4 and current local ISO timestamp
4. Rewrites two top-level frontmatter scalars **in place** (preserving all other lines byte-for-byte): `exo__Asset_uid` → new UID, `exo__Asset_createdAt` → now
5. Writes the new file as `<source-folder>/<new-uid>.md`
6. Focuses the new file in a new workspace tab via the platform's `onCreated` callback (Obsidian opens a new leaf; CLI no-op)

Uses `service_call` grounding type (no `inputSchema` — operates purely on the active file context). Used by [[5a3ebc9b-c440-4ed6-a9a9-3589eaf1c3b0|Duplicate current asset]] command (Issue #3292).
