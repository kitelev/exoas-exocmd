# workflow_transition rollback

Generic dispatcher Grounding for rollback (`isRollback=true`) workflow transitions. Resolves transition at runtime from `(assetClass, currentStatus)` against vault Workflow ABox via `GroundingExecutor.executeWorkflowTransition` (RFC 36347daf Phase 2).

Shared by Phase 1 backward Commands migrated in Phase 3:

- Re-open (Done → Doing) — postActions: clear endTimestamp + resolutionTimestamp
- Rollback to Backlog (Task: Doing → Backlog) — postActions: clear startTimestamp
- Rollback to ToDo (Project: Doing → ToDo) — postActions: clear startTimestamp
- Rollback to Analysis (Project: ToDo → Analysis) — no postActions
- Rollback to Backlog (Project: Analysis → Backlog) — no postActions
- Rollback to Draft (Backlog → Draft) — no postActions

Per-Command target class gating is via `exocmd__CommandBinding_targetClass`; per-state visibility gating is via Phase 1 `exocmd__Precondition_sparqlAsk`.
