# Has non-UID filename outside whitelist

SPARQL ASK precondition: returns `true` when the target asset's filename does not match its `exo__Asset_uid` AND the target is **not** an instance of a whitelisted label-naming class.

Whitelist (per BehavioralRule `152e39df-cc1b-4175-a4f2-2c2913018937` in vault-exodev aiKnow):

- [[b04e7a3e-6b49-4984-9f8d-b74e9f36818b]] — `pn__DailyNote` (filename `YYYY-MM-DD.md`, required by obsidian-calendar-plugin)
- [[2b754a16-cccf-49ee-af69-a18c6d1e3b63]] — `period__Week` (filename `YYYY-Www.md`, same plugin)

Used by the **Rename to UID** command ([[d0a0663b-407b-45bc-8c5c-e0ddfa883ba4]], RFC Phase 6 switch) to skip whitelist instances during bulk-rename.

VALUES IRI form — **symbolic** `<https://exocortex.my/ontology/<prefix>#<LocalName>>` per Phase 2 instances post-mortem 2026-05-22 (class assets with `<prefix>__<LocalName>` label pattern resolve to symbolic IRI for `Instance_class` target). RFC 1ce2a226 §S1/M10 (URL-encoded UUID-canon) **superseded** — STEP H empirical verification 2026-05-23 returned 0 rows for UUID-canon form vs 290 rows for symbolic DailyNote.

Whitelist expansion: edit `VALUES ?wlCls { ... }` block in this asset's frontmatter — no code change required (homoiconicity Q1).
