# Create MeetingPrototype instance

Grounding для создания `ems__Meeting` instance из `ems__MeetingPrototype`.

- `Grounding_type=create_instance` → `GroundingExecutor.executeCreateInstance`.
- `targetClass=ems__Meeting` → `exo__Instance_class: [[1b0a5e34-dd7f-4ead-b43a-6c7c5a5ecaca]]` в новом файле.
- `targetPrototype=7ab483c7-...|ems__MeetingPrototype` → `exo__Asset_prototype: [[...]]` в новом файле.
- `targetFolder=$targetFolder` — резолвится в parent folder триггер-ассета (прототипа). Новый Meeting создаётся рядом с прототипом.
- `inputSchema` — modal с полем label (Meeting name, prefill = текущая дата).

InheritanceRule: только `cbe000c4` (isDefinedBy→isDefinedBy unconditional). Принцип «prototype = virtual aggregator» — area/relates/прочие связи остаются на прототипе; instance ссылается на prototype через `exo__Asset_prototype` back-link.
