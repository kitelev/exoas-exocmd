# exocmd__Template_inheritanceRule

`exo__ObjectProperty` on `exocmd__UniversalDefaultTemplate`. Multi-valued list of `exocmd__InheritanceRule` instance wikilinks. Cardinality 0..N.

YAML serialization:
```yaml
exocmd__Template_inheritanceRule:
  - "[[<inheritance-rule-uuid-1>]]"
  - "[[<inheritance-rule-uuid-2>]]"
```

Engine merges these with Grounding's own `Grounding_inheritanceRule` list at parse-time in `CommandResolver.resolveInheritanceRules` (RFC 727572d2 Phase B). Grounding entries with matching `targetPropertyName` override Universal entries.

Universal rules apply class-conditioned: e.g. `InheritanceRule(condition: ems__TaskPrototype, source: exo__Asset_uid, target: exo__Asset_prototype)` fires only when target is a TaskPrototype instance — replaces former hardcoded `properties[backLinkProp] = ...` block в `GroundingExecutor.executeCreateInstance:710-715`.
