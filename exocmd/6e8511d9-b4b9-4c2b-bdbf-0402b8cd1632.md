# Create Class

`exocmd__Command` for the homoiconic «Create Class» button (project 85150d63,
W3). Rendered on every `exo__Ontology` definition page (via the binding
`4a1298e2-46ad-4ba9-85a5-497890244842`, `targetClass = exo__Ontology`).

Click → form (single `Label` field) → a valid UUID-named `exo__Class`
(`exo__Instance_class = exo__Class` metaclass `8619c4fc-...`,
`exo__Asset_isDefinedBy = host ontology`), co-located in the host ontology's
folder, opened.

- **Grounding**: `Create Class grounding` (`create_instance`, `targetClass =
  exo__Class metaclass` + `$targetClassSelf` isDefinedBy + `$isDefinedByFolder`).
- **Icon**: square-plus (Lucide) — distinct from Create Instance's circle-plus.
- **Category**: creation.
- No confirm dialog, no prefill defaults. Reuses 100% of the T1 «Create
  Instance» machinery (v16.102.0); only vault-declared config differs.
