# Create project

Grounding action that calls the `createRelatedProject` service to create a new
`ems__Project` with automatic parent-context inheritance.

Uses the `service_call` grounding type with `inputSchema` to collect the project
label from the user. The `createRelatedProject` handler in the plugin reads the
triggering note's metadata and writes either `ems__Effort_area` (when the parent
is an `ems__Area`) or `ems__Effort_parent` (when the parent is another Project
or Task) on the newly-created Project, mirroring the behavior of
`createRelatedTask` for child task creation.

**Plugin requirement**: v15.92.0+ (ships the `createRelatedProject` service
handler). Earlier plugin versions do not register this service and will log a
"Service not found: createRelatedProject" error on click.

**History**: this grounding previously called the generic `createAsset` handler
with `{"prototype":"ems__ProjectPrototype"}`. `createAsset` writes prototype-only
frontmatter and never inherits parent context, so the resulting Project was
orphaned from the triggering Area — no `ems__Effort_area` link, so
AssetRelations on the Area did not surface the new Project. The switch to
`createRelatedProject` is the canonical fix (symmetric with
`createRelatedTask` / `a222094b-…`).
