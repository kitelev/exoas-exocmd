# exocmd__Grounding_targetValue (DEPRECATED)

> ⚠️ **DEPRECATED** per RFC `31c1a0be` (Phase 1, 2026-05-19).
> **Do not use in new groundings.** Use one of the three disjoint replacements:
> - `[[b319e33d-2d72-4c67-a626-10db245a2667]]` — `exocmd__Grounding_targetValueRef` (for wikilink references)
> - `[[153881c3-a8c6-4103-8ee9-b5b5655a6db9]]` — `exocmd__Grounding_targetValueLiteral` (for plain strings)
> - `[[65218aaf-e092-44f4-ba51-eefd77ac0a2b]]` — `exocmd__Grounding_targetValueSubstitution` (for substitution tokens)

## History

Originally `exo__StringProperty` storing heterogeneous union (JSON-escaped wikilink / substitution token / JSON-object payload). Parser required heuristics for type dispatch. RFC `31c1a0be` reifies the three semantic categories as separate typed predicates.

## Removal plan

- **Phase 2 (migration script):** Rewrite all 65 vault grounding instances using one of the three new properties. This property no longer set on new groundings.
- **Phase 3 (parser):** Parser reads new properties; backward-compat fallback warns when legacy `targetValue` encountered.
- **Phase 5 (cleanup):** Delete this property asset entirely (after migration window confirms no legacy reads remain).
