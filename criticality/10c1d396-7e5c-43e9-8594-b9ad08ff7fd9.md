# Not in Someday zone

SPARQL ASK precondition: returns `true` when task zone is NOT Someday AND task is not in a terminal status (Done / Cancelled / Trashed / Failed). When zone is unset the zone-check passes; when status is unset the terminal-status check passes (UNBOUND `=` literal is UNDEF → OR is UNDEF → `!UNDEF` is UNDEF → filter excludes the row, but SPARQL's `!` over UNBOUND keeps the row only via the BOUND-driven first clause; in practice status is always set on `ems__Task` instances by the `Backlog` default).

Used by "Set Criticality Low" command to show the button only when relevant.
