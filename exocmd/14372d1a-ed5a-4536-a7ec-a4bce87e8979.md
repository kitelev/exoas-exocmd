# Un-archive Ontologically

The ontological **inverse** of «Archive Ontologically»
([[3de74a87-177a-4cbe-9e44-41c9df143b6c|Archive Ontologically]]). Restores an
ontologically-archived asset back into its **source** ontology and folder — a
homoiconic, single-asset, idempotent un-archival that is an *emergent composition*
of already-shipped grounding primitives. No bespoke un-archive subsystem.

Composite ([[0fa1207a-9bd3-48d1-b072-1d8447e73cc9|Un-archive Ontologically (composite)]])
of two ordered steps:

1. **Re-anchor** `exo__Asset_isDefinedBy` ([[2698e5a8-978b-4038-b1e7-f50b5fcfdb93|step 1]],
   `property_set`) — value computed by the reverse NamedQuery
   ([[514b38c1-fbde-4e79-a7b1-225075d8a779|Resolve source ontology]]) reversing
   `exo__Ontology_archiveOntology`: `?src --archiveOntology→ $currentAsset.isDefinedBy`
   (the C4 read-side `targetValueQuery` CQRS bridge).
2. **Relocate** the file ([[8e6baf1e-71dd-4d8e-8f1c-3cc69ce6e3ec|step 2]], `service_call`
   `serviceId=repairFolder`) into the source ontology's folder — the co-location
   invariant.

- **Precondition**: [[f1e97b8c-3fe3-4b9a-aa01-16512b7bb8ee|Is ontologically archived]]
  — the current `isDefinedBy` is the `exo__Ontology_archiveOntology` of some source.
  Once restored, the asset lives in the source ontology (which is nobody's archive
  target) → the ASK fails → idempotent no-op.
- **CLI**: `apply un-archive-ontologically <target>`.
- Round-trip: «Archive Ontologically» → «Un-archive Ontologically» is identity on
  `(isDefinedBy, folder)`.
