# Universal IR: ActionPrototype.uid → new.exo__Asset_prototype

When the click target is an `ems__ActionPrototype` instance, copy the target's
`exo__Asset_uid` into the new instance's `exo__Asset_prototype` — backlink
semantics. Sibling of the TaskPrototype rule (`41c28b26`); the 6th prototype-class
rule attached to the Universal Default Template singleton (`62907ff4`).

- `_sourceProperty` → `exo__Asset_uid` (`fada7446`)
- `_targetProperty` → `exo__Asset_prototype` (`22c124da`)
- `_targetClassCondition` → `ems__ActionPrototype` (`93d74dfd`)
- `_priority` → 100

So "Create Action Instance" on a BAD prototype (e.g. «Выпить Пантокальцин») writes
the new Action's `exo__Asset_prototype` pointing at that prototype.
