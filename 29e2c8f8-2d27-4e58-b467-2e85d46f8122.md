# exocmd__UniversalDefaultTemplate

RFC 727572d2. Singleton class whose instance gathers PropertyDefault and InheritanceRule entries applied to **every** create_instance Grounding.

## Properties (Phase 1)

- `exocmd__Template_propertyDefault` — multi-valued ref to `exocmd__PropertyDefault` assets. Range: PropertyDefault IRI. Cardinality 1..N.
- `exocmd__Template_inheritanceRule` — multi-valued ref to `exocmd__InheritanceRule` assets. Range: InheritanceRule IRI. Cardinality 0..N. (RFC v1 Q2 expansion — fix Bug #5 backlink semantics declaratively without 70 per-Grounding edits.)

## Singleton convention

There is **one** active UniversalDefaultTemplate ABox instance per vault. Engine looks up the asset whose `exo__Instance_class` includes this class — if multiple are present, engine selects deterministically by `exo__Asset_uid` lexicographic order with a warn log.

## Engine semantics (parse-time merge)

`CommandResolver.resolvePropertyDefaults` (RFC 727572d2 Phase B):
1. Load Universal Template's `_propertyDefault` list (cached).
2. Load Grounding's `_propertyDefault` list.
3. Merge: Grounding entries override Universal by `propertyName` key.
4. Return merged `PropertyDefaultResolved[]` to executor.

`CommandResolver.resolveInheritanceRules` (analogous): merge by `targetPropertyName`.

Safety net: if Universal Template absent, engine falls back to legacy TS primitives in `GroundingExecutor.executeCreateInstance` with `logger.error` flagging unhealthy state.
