# Archive Ontologically → Asset binding

Binds the "Archive Ontologically" command (`3de74a87`) to all `exo__Asset`
instances (inline, `maintenance` group). Universal scope — the inline button
appears on every asset whose precondition
[[9b0fdc10-ccb9-46b0-80b8-d9a250a01c9b|Archived and not yet ontologically archived]]
matches (`archived: true` **and** the asset's ontology declares an
`exo__Ontology_archiveOntology`).

Created 2026-07-11: the "Archive Ontologically" command shipped (RFC 78c2b7d0
C5) with a precondition + composite grounding but **no CommandBinding** — so
`resolveForAssetMulti` never surfaced it and the inline button never rendered,
even though `apply archive-ontologically` (cliName → `loadCommand`, no binding
needed) worked. Directly parallels the sibling `58548853` note (Archive
Completed had the identical missing-binding bug, restored 2026-04-26).

`targetClass: exo__Asset` mirrors the Archive Completed sibling (broadest scope;
the precondition already gates applicability). `order: 910` — placed just after
Archive Completed (`900`) within the maintenance group.
