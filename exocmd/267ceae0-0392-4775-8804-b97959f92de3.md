# Create Next Iteration

Homoiconic by-class command re-bound to `ems__Task` (binding `c20d2304`, req
`2d1ffced`). For a task you have **finished** and will run again next cycle:
spawns a fresh copy of it in your Backlog, carrying all context forward, linked
back to the iteration it followed. The source Done task is **not** touched.

Fires the `create_instance` grounding `d240c457`, which clones the clicked task
into a new **Backlog** `ems__Task` (label / body / area / parent / zone / relates /
deadline / estimate carried forward via the shared «Partially Done» InheritanceRule
mechanism), with `ems__Effort_prevIteration` → the finished task. The new task gets
a fresh uid/createdAt and inherits neither execution timestamps nor
`ems__Effort_partiallyDone`; it co-locates with the original (`$targetFolder`).

- **Precondition**: `4090169b` — visible only when the target is in status **Done**
  AND not archived AND not a prototype (reuses «Re-open»'s Done+not-prototype gate
  `2bee97b7` AND the archived-guard `28f722ba`). Naturally hidden in Backlog /
  Doing / Archived.
- **Grounding**: `d240c457` (`create_instance`, modernized). **Category**: status.
  **Icon**: plus-square. **CLI**: `create-next-iteration`.
- **Difference from «Partially Done»** (Doing-only, «failed → retry» — additionally
  closes the source done-but-partial) and «Следующая итерация» (WaitingCheckTask):
  Create Next Iteration is the *success* path — it never mutates the finished task,
  only spawns the next cycle.
