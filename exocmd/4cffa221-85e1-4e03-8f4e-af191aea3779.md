# Has UID filename

Host-function precondition: returns `true` when the target asset's filename equals its `exo__Asset_uid` (UUID-canon, per CLAUDE.md TBox rule).

Inverse of [[b610b0e5-beb2-43b0-b2fd-1601c7a5d160|Has non-UID filename]] (whose whitelist-aware variant [[e2759bc2-a102-4b14-ab9c-662ee0e1a642|Has non-UID filename outside whitelist]] gates the "Rename to UID" command).

Implementation: `packages/exocortex/src/services/preconditionHostFunctions.ts:hasUidFilename` — synchronous, returns `false` when:
- the context has no `assetUid` (file has no `exo__Asset_uid` in frontmatter)
- the file basename does not equal the asset's UID (label-named files: `pn__DailyNote` daily notes `YYYY-MM-DD.md`, `period__Week` weekly notes `YYYY-Www.md`, freeform notes)

Synchronous evaluation is mandatory for Obsidian's Command Palette `checkCallback` — see `ExocmdCommandPaletteRegistrar.registerActiveTargetCommand` and `PreconditionEvaluator.evaluateHostFunctionSync`.

Used by [[5a3ebc9b-c440-4ed6-a9a9-3589eaf1c3b0|Duplicate current asset]] command (Issue #3292) to hide the palette entry when the active file isn't a UUID-canon Exocortex asset (avoids creating a `<new-uid>.md` duplicate of a daily note, which would break calendar-plugin recognition).
