# $grounding.targetClass

`exocmd__SubstitutionToken` instance. Resolves at execution time to the active Grounding's `targetClass` value (resolved to UID-canon, e.g. `"[[1b20a8f0-d745-4e93-91db-4531b3df120e]]"` for `ems__Task`). Parameterless — context resolver reads from the active GroundingDefinition.

Added per RFC 727572d2 Q1 decision: Universal Default Template carries a single PropertyDefault for `exo__Instance_class → $grounding.targetClass`, replacing the per-Grounding migration alternative (20 separate PD assets).

Resolver implementation reads from execution context (Grounding instance metadata available to executor); CommandResolver may pre-resolve at parse-time if class label-to-UID resolution is reachable.
