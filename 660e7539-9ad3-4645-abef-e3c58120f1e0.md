# exocmd__SubstitutionTokenLegacy

> ⚠️ **DEPRECATED** — superseded by [[08cec529-90eb-4d43-88de-ceecccea12b0|exocmd__SubstitutionToken]] (RFC v2 Phase 1, 2026-05-21).
>
> Renamed from `exocmd__SubstitutionToken` to disambiguate label collision with the new RFC v2 class (`08cec529`). Both classes coexist while RFC v2 migration is in flight; this one will be removed when Phase 4+ deprecates `exocmd__Grounding_targetValueSubstitution` and the legacy `GroundingExecutor.substituteVariables` handler.

Класс substitution-token'ов — placeholder'ов, resolved'имых GroundingExecutor в момент выполнения grounding'а.

## Использование (legacy path)

Используется как range property `exocmd__Grounding_targetValueSubstitution`. Когда grounding указывает `targetValueSubstitution: "[[<token-uid>]]"`, parser смотрит метку токена и подставляет соответствующее значение в `targetValue` перед записью в frontmatter.

Implementation: `packages/exocortex/src/services/GroundingExecutor.ts` — `substituteVariables`.

## Замыкание мира (closed-world)

Class sealed — добавление нового token требует RFC. Все resolver-supported tokens должны иметь corresponding instance в vault.

Source RFC: `31c1a0be-bc72-4a1e-8426-75e7b0ddedd2` (vault-exodev/inbox — RFC: Grounding as first-class RDF).

## Successor class

The RFC v2 replacement is [[08cec529-90eb-4d43-88de-ceecccea12b0|exocmd__SubstitutionToken]] — same label-namespace purpose but reparented under `exocmd__DeclarativeRule` (homoiconic Grounding rules hierarchy). Its instances carry `exocmd__SubstitutionToken_name` (xsd:string, unique) + `_resolver` properties instead of the legacy enum-membership-only model. Migration scheduled for RFC v2 Phase 4+ (#3166).
