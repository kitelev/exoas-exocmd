# exocmd__Command_openInSameTab

`exo__BooleanProperty` на `exocmd__Command` (cardinality 0..1, range `xsd:boolean`, default `false`).

Opt-in флаг для Command-ов, которые порождают новый файл (через `Grounding_type=create_instance`) — управляет тем, как plugin открывает новый файл после создания:

- **`true`** → `ObsidianFileOpener.open(path, { sameTab: true })` → `app.workspace.getLeaf(false)` → навигация в **текущей** активной вкладке.
- **`false` / отсутствует** → `getLeaf("tab")` → новая вкладка (legacy / default behavior).

## Background

Добавлено в RFC `ce27e55d-e5e6-4b52-86ba-3ac9b30de38a` (одноразовое расширение TBox interpretation infrastructure) — после этого любая supervision-style кнопка в RDF может декларативно опт-инить same-tab behavior без правки кода плагина.

Plugin PR: [kitelev/exocortex#3287](https://github.com/kitelev/exocortex/pull/3287) (merged 2026-05-29 как `de9ae418`).

## Semantics

- Применяется к **любому** Command, не только create_instance — но эффект виден только когда `Grounding.openPath` определён (т.е. был создан файл).
- IFileOpener consumer (Obsidian plugin) — единственный consumer; CLI/headless игнорируют `opts.sameTab`.
- Forward path: `CommandExecutionFlow.run()` передаёт `command.openInSameTab` в `fileOpener.open(path, { sameTab })` после успешного `execute()`.
